ssize_t SH_ParserParse ( SH_Parser *  parser,
char *  buf 
)

Parses a character stream, creates tokens, and generates command statements.


This step performs grouping of tokens based on the following production rules:



node: comment | command | io_redir | bg_ctrl
comment: ^((whitespace)* '#' whitespace)
command: ^(word)+ [excluding '<', '>', '&', '#', whitespace, newline]
bg_ctrl: (whitespace '&' (whitespace | newline))$
io_redir: whitespace ('<' | '>') whitespace word
word: any consecutive characters, excluding whitespace and newline
whitespace: (' ' | '\t')+
newline: '\n'

Parameters
parserParser object
bufcharacter stream to parse
Returns
number of statements created on success, -1 on failure
Note
Caller is responsible for freeing parsed statements via SH_DestroyStatement.

char* SH_ParserSubstituteVariable ( char *  word,
char **  old_ptr,
char **  new_ptr,
size_t *  len

Substitutes variable followed by '$' into word and returns result. If no valid variable follows '$', then the '$' is copied into word instead.

This is used during the expansion step of the parser, where variable substitution is performed. As of now, only PID expansion is supported.

Precondition
old_ptr must point to the first '$' character.
new_pointer must point to the first null ('\0) character in word.
len must be long enough to fit original string characters plus null terminator.
Parameters
wordthe word string to extend
old_ptrpointer to '$' character in original string
new_ptrpointer to next available character in word for insertion
leninput/output parameter for final string length
Returns
new string with variable substituted at end, or just the '$' if no valid variables present
